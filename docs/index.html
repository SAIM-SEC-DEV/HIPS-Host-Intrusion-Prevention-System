Verified Installation
Follow these steps in order
1

Database Setup

Create the database and import the schema.

CREATE DATABASE hips_db;
USE hips_db;
SOURCE database/schema.sql;
ℹ️ Optional: The fourth migration step (database/migration.sql) was not required during testing and may be skipped unless additional updates are provided.
2

Server Setup

Copy the entire server folder into XAMPP's htdocs directory.

xampp/
└── htdocs/
    └── server/          ← folder name must remain "server"
        ├── dashboard/
        ├── api/
        ├── config/
        └── ...
ℹ️ Apache serves only files inside htdocs. The folder name server is required because the agent configuration references it.
3

Agent Configuration

Rename the example config and set the required values.

// Rename this file:
config/hips-agent.properties.example
// To:
config/hips-agent.properties
server.url         = http://localhost/server
auth.token         = # automatically generated — do not modify
agent.owner        = Office-PC
watch.directories  = C:\Users\<username>\Desktop
ℹ️ auth.token is automatically generated. Do not change it. agent.owner can be any descriptive name.
4

Recompile Agent

Run the recompilation script to apply the new configuration.

recompile-agent.bat
💡 Recompilation updates the agent binary with your settings before execution.
⚠️ The generated dist/ folder is intended for portable deployment. During the tested setup, the agent is executed from the original project using run-agent.bat, not from dist/.
5

Run the Server

Start all server components with a single script.

Start HIPS Server.bat
ℹ️ This starts Apache, MySQL, the dashboard, and the server API. After startup, access the dashboard at http://localhost/server/dashboard.
6

Run the Agent

Start the endpoint monitoring agent from the original project directory.

run-agent.bat
✅ This script uses the compiled agent from the project root — not the dist/ folder — and begins monitoring according to your hips-agent.properties.

Included Scripts
Start HIPS Server.bat
.bat · Windows Batch
Starts Apache, MySQL, HIPS Dashboard, and Server API automatically.
stop-hips-server.bat
.bat · Windows Batch
Gracefully stops all HIPS server services.
run-agent.bat
.bat · Windows Batch
Runs the compiled monitoring agent from the original project directory (not from dist/).
discovery-launcher-temp.bat
.bat · Windows Batch
Automatic discovery launcher for agent deployment across the network.
discovery-service.bat
.bat · Windows Batch
Discovery service for multi‑machine deployments. Coordinates agent registration across endpoints.
hips-watchdog-temp.ps1
.ps1 · PowerShell
Monitors running HIPS processes and auto‑recovers from crashes.
  • Detects service crashes
  • Automatically restarts failed components
  • Improves overall reliability
deploy-package.bat
.bat · Windows Batch
Packages a deployment‑ready distribution for additional endpoints. Output includes agent binaries, config templates, and startup scripts.
