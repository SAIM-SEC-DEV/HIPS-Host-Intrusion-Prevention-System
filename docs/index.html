Quick Start Recommended
For users who just want to run HIPS on a single machine
Requires XAMPP
Requires Java 17+
1

Copy the HIPS Project into XAMPP

Copy the entire HIPS project folder (the folder containing the dashboard, API, and related files) into your XAMPP htdocs directory.

XAMPP/
└── htdocs/
    └── hips/
        ├── dashboard/
        ├── api/
        ├── config/
        ├── agent/
        └── ...
ℹ️ Why this step is needed: Apache (via XAMPP) only serves files placed inside the htdocs directory. Since HIPS is a PHP-based dashboard and API, the project must be placed there for Apache to serve it correctly. Otherwise, visiting http://localhost/hips will result in a "Page Not Found" or similar error.
2

Configure the Agent

Rename the example config file, then open it and fill in your values:

// Rename this file:
config/hips-agent.properties.example

// To:
config/hips-agent.properties
server.url         = http://localhost/hips
auth.token         = YOUR_TOKEN
agent.uuid         = GENERATE_UUID
agent.owner        = Workstation-01
watch.directories  = C:\Users\<username>\Desktop
ℹ️ Generate a UUID at uuidgenerator.net — each endpoint must have a unique value.
3

Start the Agent

run-agent.bat

The agent will connect to the server and begin endpoint monitoring.

4

Start the Server

Run the startup script from the project root. This starts Apache, MySQL, and initialises the HIPS server automatically.

start-hips-server.bat
✅ This script handles Apache, MySQL, and HIPS dashboard startup in a single step.
2
Multi-Machine Deployment
For organizations or lab environments with multiple endpoints

One central server hosts the dashboard and database. Each endpoint runs an agent pointing back to the server IP. Full guide at docs/multi-machine-guide.md.

Server PC │ ├── XAMPP (Apache + MySQL) ├── MySQL (hips_db) └── Dashboard → http://<SERVER-IP>/hips/dashboard Agent PC 1 → server.url=http://192.168.1.50/hips Agent PC 2 → server.url=http://192.168.1.50/hips Agent PC 3 → server.url=http://192.168.1.50/hips
ℹ️ Every agent needs a unique agent.uuid and the same auth.token as configured on the server.

Configuration Reference
Property Description Example
server.url URL of the HIPS dashboard server http://192.168.1.50/hips
auth.token Shared secret between agent and server. 16+ chars, mixed case + symbols. 6j8J#v9Lm2@Xp83Rt
agent.uuid Unique machine identifier. Generate at uuidgenerator.net 1f32f7d8-7c8c-4f8c-a431-c3f1dbe93c77
agent.owner Human-readable label shown in the dashboard Finance-PC · SOC-Lab-01
watch.directories Comma-separated list of paths to monitor for file changes C:\Users\John\Desktop,C:\Users\John\Documents
watch.directories — Examples
// Desktop only
watch.directories = C:\Users\<username>\Desktop

// Desktop + Documents
watch.directories = C:\Users\<username>\Desktop,C:\Users\<username>\Documents

// Custom sensitive folder
watch.directories = D:\SensitiveData
Included Scripts
start-hips-server.bat
.bat · Windows Batch
Starts all server-side services in one step.
  • Starts Apache
  • Starts MySQL
  • Initialises the HIPS dashboard
stop-hips-server.bat
.bat · Windows Batch
Gracefully stops all HIPS server services.
run-agent.bat
.bat · Windows Batch
Starts the endpoint monitoring agent using the pre-compiled binary in agent/bin.
discovery-launcher-temp.bat
.bat · Windows Batch
Automatic discovery launcher for agent deployment across the network.
discovery-service.bat
.bat · Windows Batch
Discovery service for multi-machine deployments. Coordinates agent registration across endpoints.
hips-watchdog-temp.ps1
.ps1 · PowerShell
Monitors running HIPS processes and auto-recovers from crashes.
  • Detects service crashes
  • Automatically restarts failed components
  • Improves overall reliability
powershell -ExecutionPolicy Bypass -File hips-watchdog-temp.ps1
deploy-package.bat
.bat · Windows Batch
Packages a deployment-ready distribution for additional endpoints. Output includes:
  • agent/ compiled binaries
  • config/ property templates
  • Required libraries
  • Startup scripts
